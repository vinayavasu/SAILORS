CRITICAL FINDINGS
------------------------------------------------------------------------------
  PRAX-2026-07-21-001  Admin plane (goals, config, invoice review, vendor
            trust) has no authentication of any kind
      Action 1: Add an authentication/authorization check (session-based admin
                login or API key) that runs before every route in admin_bp,
                not just the ones that look sensitive.
      Action 2: Specifically gate update_finbot_goals and update_finbot_config
                behind that check first, since they alter the agent's own
                security posture.

  PRAX-2026-07-21-002  Invoice approval has no server-side enforcement of the
            manual-review threshold
      Action 1: Add a hard, independent check immediately before any commit in
                _approve_invoice: if invoice.amount >
                config.manual_review_threshold, reject the approval path
                entirely regardless of confidence, reasoning, or
                business_context score.
      Action 2: That gate must not be expressible as an LLM tool-call outcome
                - it should run in code the model cannot talk its way around.

  PRAX-2026-07-21-003  Unauthenticated goals endpoint injects
            operator-supplied text with override priority into the system
            prompt
      Action 1: Require authenticated, role-gated access before any
                custom_goals write (shared root cause with
                PRAX-2026-07-21-001).
      Action 2: Even for authorized updates, do not let injected goal text
                override baseline security-relevant behavior (fraud detection,
                thresholds) - separate 'business preference' goals from
                'security control' goals structurally so one cannot override
                the other.

HIGH FINDINGS
------------------------------------------------------------------------------
  PRAX-2026-07-21-004  A detected prompt injection can still be auto-approved
            if business-context scoring is high enough
      Action: When contains_prompt_injection is true, remove every code path
              that can still resolve to auto_approve; route unconditionally to
              human review regardless of business_context or speed_priority.

  PRAX-2026-07-21-005  No persisted audit trail for any security-relevant
            admin action
      Action: Add structured, persisted audit logging for update_finbot_goals,
              update_finbot_config, review_invoice, and update_vendor_trust:
              who changed what, previous and new values, and a timestamp,
              written somewhere durable - not console print().
